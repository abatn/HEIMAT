# HEIMAT – Fortschritt

## ARCHITEKTUR-ANALYSE (aus MD-Dateien + Fehleranalyse)

### Daten-Pipeline
```
PostgreSQL (DECIMAL) → Node.js pg (String) → JSON (String) → Flutter (.toDouble() scheitert)
```

### Ursache
- PostgreSQL: `latitude DECIMAL(10, 8)` gibt Strings zurück
- Node.js pg: Gibt DECIMAL-Werte als Strings in JSON
- Flutter: Erwartet Numbers, bekommt Strings

### Korrekte Lösung
**Nicht** nur `.toDouble()` fixen, sondern:
1. **Backend:** Daten-Typen in PostgreSQL korrekt definieren
2. **Oder:** Frontend: Strings robust parsen mit `double.parse()`
3. **Oder:** API: Daten im Backend konvertieren

### Empfehlung
**Option 2 (Frontend-Fix)** ist am einfachsten:
- `double.parse(json['latitude'].toString())` statt `(json['latitude'] ?? 0).toDouble()`
- Das ist der korrekte Weg weil PostgreSQL DECIMAL immer als String kommt
