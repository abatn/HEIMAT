# Warum Open Source?

## Die Philosophie hinter HEIMAT 2.0

Open Source ist mehr als nur eine Lizenz. Es ist eine Haltung. Bei HEIMAT 2.0 ist Open Source das Fundament für Vertrauen, Transparenz und gemeinschaftliches Schaffen.

## Was bedeutet Open Source?

### Die Grundidee
Open Source bedeutet: Der Quellcode eines Projekts ist öffentlich einsehbar. Jeder kann ihn lesen, verändern und weitergeben. Das gilt auch für HEIMAT 2.0.

### Die vier Freiheiten (Definition der FSF)
1. **Freiheit 0:** Das Programm beliebig ausführen
2. **Freiheit 1:** Das Programm studieren und verändern
3. **Freiheit 1:** Kopien verteilen
4. **Freiheit 2:** Modifizierte Versionen verbreiten

## Warum ist das wichtig?

### 1. Vertrauen durch Transparenz
Wenn der Code öffentlich ist, kann jeder prüfen:
- Welche Daten werden gesammelt?
- Wie werden sie verarbeitet?
- Gibt es versteckte Funktionen?

Bei proprietären Apps müsst ihr dem Entwickler vertrauen. Bei Open Source könnt ihr selbst prüfen.

### 2. Kein Vendor Lock-in
Kommerzielle Apps können:
- Den Service einstellen
- Die Preise erhöhen
- Die Regeln ändern

Bei Open Source:
- Der Code bleibt erhalten
- Die Community kann fortführen
- Ihr seid unabhängig

### 3. Gemeinschaftliche Entwicklung
Open Source lebt von der Community:
- Fehler werden schneller gefunden
- Features werden gemeinsam entwickelt
- Wissen wird geteilt

### 4. Kostenfreiheit
Keine Lizenzgebühren, keine Abos, keine versteckten Kosten. Open Source ist oft kostenlos – und das bleibt es auch.

## Open Source bei HEIMAT 2.0

### Unsere Tools
| Komponente | Lizenz |
|------------|--------|
| Flutter | BSD-3 |
| Node.js | MIT |
| PostgreSQL | PostgreSQL-Lizenz |
| OpenStreetMap | ODbL |
| GNU Taler | AGPL-3.0 |
| Matrix | AGPL-3.0 |

### Unsere Lizenz: AGPL-3.0
Wir haben uns für die AGPL-3.0 entschieden:
- **Stärkste Open-Source-Lizenz** für SaaS
- **Verhindert Proprietarisierung** durch Cloud-Anbieter
- **Sicherstellt**, dass Änderungen öffentlich bleiben

### Transparenz in der Praxis
- **GitHub:** Jeder kann Issues erstellen und PRs einreichen
- **Dokumentation:** Alle Entscheidungen werden dokumentiert
- **Discussions:** Offene Diskussionen über die Zukunft

## Open Source vs. Proprietär

| Aspekt | Open Source | Proprietär |
|--------|-------------|------------|
| **Transparenz** | Volle Einsicht | Keine Einsicht |
| **Kosten** | Kostenlos | Oft teuer |
| **Kontrolle** | Community | Unternehmen |
| **Zukunft** | Sicher | Unsicher |
| **Datenschutz** | Prüfbar | Vertrauenssache |

## Herausforderungen

### 1. Funding
Open Source braucht Geld für:
- Server und Hosting
- Entwicklungszeit
- Community-Management

**Lösung:** Spenden, Förderungen, Sponsoring

### 2. Community-Aufbau
Eine aktive Community ist essenziell:
- Entwickler finden
- Beiträge koordinieren
- Konflikte lösen

**Lösung:** Gute Dokumentation, CONTRIBUTING.md, Matrix-Room

### 3. Qualitätssicherung
Ohne kommerziellen Druck:
- Wer testet?
- Wer reviewt?
- Wer verantwortet?

**Lösung:** CI/CD, Code Reviews, Tests

## Wie kann ich mitmachen?

### Als Nutzer
- Testet die App
- Gebt Feedback
- Meldet Bugs

### Als Entwickler
- Schaut euch die Good-First-Issues an
- Macht euch mit CONTRIBUTING.md vertraut
- Schickt Pull Requests

### Als Organisation
- Setzt HEIMAT ein
- Werdet Sponsor
- Helft bei der Verbreitung

## Fazit

Open Source ist mehr als eine Technik. Es ist eine Grundhaltung. Bei HEIMAT 2.0 glauben wir daran, dass Technologie der Gesellschaft gehört – nicht umgekehrt.

**Mitmachen erwünscht. Code ist Rede.**
